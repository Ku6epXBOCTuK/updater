# Launcher-Core

Универсальная система самообновления для портативных Windows-приложений.

## Описание

Launcher-Core — это набор инструментов для создания портативных Windows-приложений с механизмом прозрачного самообновления. Система полностью абстрагирована от конкретного ПО и позволяет собирать брендированные лаунчеры из единого исходного кода.

### Компоненты системы

- **Launcher Source**: Репозиторий с исходным кодом на Rust, содержащий шаблон лаунчера и утилиту `sign-tool` для работы с манифестами и ключами
- **Sign-Tool**: CLI-утилита для генерации ключей Ed25519 и создания подписанных манифестов для ZIP-архивов с обновлениями

### Возможности

- **Автоматическое обновление**: Прозрачное обновление приложения без вмешательства пользователя
- **Безопасность**: Использование алгоритма Ed25519 для подписи манифестов и SHA256 для проверки целостности
- **Атомарные обновления**: Гарантия, что обновление либо полностью применится, либо не затронет работающую версию
- **Офлайн-режим**: Приложение продолжает работать при отсутствии сети, если локальная версия исправна
- **Логирование**: Детальное логирование всех этапов обновления с ротацией логов

## Установка

```bash
cargo build --release
```

После сборки исполняемые файлы будут находиться в директории `target/release/`.

## Использование

### Генерация ключей

```bash
# Создать ключи в папке ./keys
sign-tool generate-keys --out-dir ./keys

# Принудительно перезаписать существующие ключи
sign-tool generate-keys --out-dir ./keys --force
```

### Подписание архива

```bash
sign-tool sign \
  --package ./release/Package.zip \
  --url "https://example.com/Package.zip" \
  --version "1.2.3" \
  --private-key ./keys/private_key.pem \
  --output ./release/update_manifest.json
```

## Архитектура

Система состоит из двух основных компонентов:

1. **Лаунчер** — точка входа в приложение, которая проверяет наличие обновлений и загружает их
2. **Sign-Tool** — утилита для создания подписанных манифестов обновлений

### Формат манифеста

Манифест (`update_manifest.json`) — JSON-объект:

```json
{
  "version": "1.2.3",
  "url": "https://example.com/download/Package.zip",
  "sha256": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
  "signature": "ed25519_base64_подпись"
}
```

Подпись формируется путём подписывания канонической строки `<version>|<url>|<sha256>` приватным ключом.

### Параметризация при компиляции

В код лаунчера через переменные окружения внедряются:

- `APP_NAME` — имя приложения (для имён файлов)
- `MANIFEST_URL` — постоянная ссылка на `update_manifest.json` (HTTPS)
- `PUBLIC_KEY` — публичный ключ Ed25519 в hex-формате

## Безопасность

- **Ed25519** — гарантия авторства манифеста
- **SHA256** — контроль целостности загружаемого архива и локального ядра
- **HTTPS** — обязателен для всех URL для предотвращения MITM-атак
- **Атомарные переименования** — исключают повреждение приложения при сбоях
- **Блокировка процессов через мьютекс** — предотвращает замену файлов во время работы ядра
